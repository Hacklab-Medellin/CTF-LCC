Reindexing Explaination
Over time it is possible for search indexes to get out of date. The script will do everything it can to maintain the index on it's own, but there can still be cases where things can get out of sync. This page gives you the option of recreating the search index on your site, this will fix any unexpected search results, or problems by recreating all of or part of the index based on the criteria you choose above. PLEASE NOTE: *ALL ACTIVE LISTINGS* AND *ALL FIELDS* COULD TAKE SEVERAL MINUTES TO RUN DEPENDING ON THE SIZE OF YOUR SEARCH INDEX AND THE SPEED OF YOUR SERVER, PLEASE BE PATIENT DURING THE PROCESS. ON AVERAGE YOU CAN EXPECT 100-200 LARGE LISTINGS TO BE INDEXED PER MINUTE

"Items" - This will allow you to reindex an individual listing, a group of listings, or the entire site. (Only currently "Active" items are listed and are sorted alphabetically by title)

"Fields" - Index only specified fields. 'All Fields' will index the whole listing. 'Main Fields' will index only the parts like Title, Description, Added Description, City, and State. The Custom Options will allow you to reindex only the custom fields on the listing.

"Items + Fields" - Reindex only specified fields in the selected item.

"Indexed Terms Per-Refresh" - This allows you to control how many TERMS (not Listings, but the individual Terms in the Listings) are indexed before PHP refreshes the page to prevent the server from timing out. The faster the server the higher the number can be. 7500 is usually a safe number for a site that has less than 1000 listings. If you are getting timeout errors or redirected to a blank page then the number is too high, if you are getting a browser error saying that there have been too many redirects then the number is too low. If your site is so large that you can't find a good balance then you will need to increase your "max_execution_time" in your PHP config.

This refresh is only used when "All Active Items" is the option selected, it is ignored when indexing a single or range of items.

"Truncate Index" - If you want to delete the entire index >>Click Here<< . (This is useful if you are switching servers or moving the database for some reason, just truncate the index here and then reindex the site once the new database is up. This will save you some time from having to export and import the index in the new database.)
